## 1. 项目概述

这是一个**基于网页的 AR 文化展示平台**，用户用手机扫描二维码或打开网址，将摄像头对准标记图片，就能看到 3D 模型叠加在现实世界中。

### 核心功能

| 功能 | 说明 |
|------|------|
| 📱 AR 扫描 | 手机摄像头对准标记图，浮现 3D 模型 |
| 🖥️ 管理后台 | 新增/编辑/删除 AR 内容，上传模型和标记图 |
| 📷 二维码 | 每个 AR 内容自动生成二维码，可打印张贴 |
| 🎨 四大分类 | 农特产品、文化景点、非遗手艺、乡村风貌 |

### 技术架构

```
浏览器（手机/电脑）
    ↓
Node.js 服务器（Express）
    ↓
SQLite 数据库（单文件，无需额外安装数据库软件）
```

- **AR 引擎**: mindAR（图像识别追踪）
- **3D 渲染**: Three.js
- **后端**: Node.js + Express
- **数据库**: SQLite（文件型数据库，自动创建）

## 2. 准备工作

### 2.1 需要的环境

在电脑上安装以下软件：

- 下载地址：https://nodejs.org

- 下载地址：https://git-scm.com/downloads

## 3. 本地运行

### 3.1 首次启动

```bash
# 1. 进入项目目录
cd tangkou-ar

# 2. 安装依赖（仅首次需要，约1-2分钟）
npm install

# 3. 初始化数据库和示例数据

npm start
```

## 4. 管理后台操作

### 4.1 登录

1. 打开 `http://localhost:3000/admin`
2. 输入管理员账号密码（默认：`admin` / `tangkou2024`）
3. ⚠️ **上线前务必修改默认密码！**（见 [7.1 修改管理员密码](#71-修改管理员密码)）

### 4.2 管理分类

进入"分类管理"页面，可以：
- 查看现有四个分类（农特产品、文化景点、非遗手艺、乡村风貌）
- 新增分类
- 编辑分类名称和图标
- 调整分类排序

### 4.3 管理 AR 内容

进入"AR 内容管理"页面，可以：

1. **新增内容**：填写标题、简介、选择分类
2. **上传文件**：为每个内容上传以下文件：

   | 文件类型 | 要求 | 必需 |
   |----------|------|------|
   | 标记图片 | JPG/PNG，纹理丰富，≥512×512px | AR 模式必需 |
   | 3D 模型 | GLB/GLTF 格式（用 Blender 导出） | 是 |
   | 缩略图 | JPG/PNG，列表展示用 | 推荐 |
   | 语音导览 | MP3 格式 | 可选 |
   | 详细介绍 | 自动生成或手动编辑 HTML | 推荐 |

3. **编译标记图**：上传标记图片后，点击"编译"按钮自动生成 `.mind` 特征文件
4. **生成二维码**：点击"生成二维码"，下载 PNG 图片用于打印张贴
5. **发布/下架**：控制内容是否在首页显示

### 4.4 查看和导出二维码

进入"二维码管理"页面，可以：
- 查看所有已生成二维码的内容
- 下载单个二维码
- 批量生成未生成二维码的内容

## 5. 制作 AR 内容

### 5.1 准备 3D 模型

推荐使用 **Blender**（免费开源）制作 3D 模型：

1. 下载 Blender：https://www.blender.org
2. 建模或导入现有模型
3. 导出为 **GLB 格式**（文件 → 导出 → glTF 2.0）
4. 文件大小建议 < 20MB，过大的模型可以在管理后台上传时自动压缩

### 5.2 准备标记图片

标记图片是 AR 识别的基础，**图片质量直接影响识别效果**。

**推荐标准：**
- ✅ 纹理丰富、细节清晰（如海报、包装盒图案）
- ✅ 正方形或接近正方形
- ✅ 分辨率 ≥ 512×512 像素
- ❌ 避免大面积纯色
- ❌ 避免对称图案（如圆形、正方形边框）
- ❌ 避免文字为主的图片

**编译标记图：**

上传标记图后，在管理后台点击"编译"按钮即可自动生成 `.mind` 特征文件。

也可以手动编译：
- 在线工具：https://hiukim.github.io/mind-ar-js-doc/tools/compile
- 命令行：`npx @mindar/image-target-compiler -i input.png -o output.mind`

### 5.3 实物落地流程

```
1. 制作3D模型（Blender → .glb）
       ↓
2. 制作/选择标记图片（如农产品包装、景点海报）
       ↓
3. 在管理后台上传模型 + 标记图 → 编译
       ↓
4. 生成二维码 → 下载打印
       ↓
5. 将二维码和标记图张贴到实物/景点旁
       ↓
6. 用户扫码 → 对准标记图 → AR 呈现！
```

## 6. 部署到云服务器

### 6.1 方案一：Docker 部署（推荐）

项目已包含 `Dockerfile` 和 `.dockerignore`，可直接在任何支持 Docker 的服务器上运行。

**步骤：**

```bash
# 1. 将项目上传到服务器（或从 Git 拉取）
scp -r tangkou-ar/ user@你的服务器IP:/home/user/

# 2. SSH 登录服务器
ssh user@你的服务器IP

# 3. 进入项目目录
cd /home/user/tangkou-ar

# 4. 复制环境变量模板并编辑（务必修改 JWT_SECRET 和管理员密码！）
cp .env.example .env
nano .env
# 必须修改以下三项：
#   JWT_SECRET=改成随机字符串（如乱敲一串字母数字）
#   ADMIN_PASSWORD=改成强密码（至少8位，建议字母+数字）
#   BASE_URL=http://你的服务器IP:3000  （部署后用实际域名替换）

# 5. 构建 Docker 镜像（.dockerignore 会自动排除 node_modules、.env 等）
docker build -t tangkou-ar .

# 6. 首次启动前先初始化数据库
mkdir -p data
docker run --rm -v $(pwd)/data:/app/data --env-file .env tangkou-ar node server/utils/seed.js

# 7. 启动容器（后台运行，开机自启）
docker run -d \
  --name tangkou-ar \
  --restart always \
  -p 3000:3000 \
  -v $(pwd)/data:/app/data \
  --env-file .env \
  tangkou-ar
```

**常用 Docker 命令：**

| 命令 | 作用 |
|------|------|
| `docker ps` | 查看运行中的容器 |
| `docker logs tangkou-ar` | 查看日志 |
| `docker restart tangkou-ar` | 重启服务 |
| `docker stop tangkou-ar` | 停止服务 |

### 6.2 方案二：直接运行（需安装 Node.js）

```bash
# 1. 服务器上安装 Node.js（同 2.1 节）

# 2. 安装 PM2 进程管理器（保持后台运行）
npm install -g pm2

# 3. 启动项目
npm install
npm run seed
pm2 start server/index.js --name tangkou-ar

# 4. 设置开机自启（Linux/Mac 可能需要 sudo）
pm2 save
sudo pm2 startup   # 按提示复制并执行显示的命令
```

**常用 PM2 命令：**

| 命令 | 作用 |
|------|------|
| `pm2 list` | 查看所有进程 |
| `pm2 logs tangkou-ar` | 查看日志 |
| `pm2 restart tangkou-ar` | 重启 |
| `pm2 stop tangkou-ar` | 停止 |

### 6.3 方案三：Render.com 一键部署（免费额度）

项目已有 `render.yaml` 配置：

1. 在 GitHub 上创建仓库，推送代码
2. 在 [render.com](https://render.com) 注册账号
3. 连接 GitHub 仓库
4. Render 自动识别 `render.yaml`，点击部署即可
5. 免费额度：每月 750 小时运行时间

### 6.4 配置域名（可选）

如果购买了域名（如 `www.tangkou-ar.cn`）：

1. 在域名服务商处添加 A 记录，指向服务器 IP
2. 安装 nginx 反向代理：

```nginx
server {
    listen 80;
    server_name www.tangkou-ar.cn;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

3. 修改 `.env` 中 `BASE_URL=https://你的域名`
4. 重启服务

## 7. 日常维护

### 7.1 修改管理员密码

**方式一：命令行（推荐）**

```bash
node scripts/change-password.js 新密码
# 密码要求：至少8个字符，建议包含字母和数字
```

**方式二：登录管理后台**

进入后台 → 修改密码

### 7.2 备份数据

**只需备份两个位置：**

```bash
# 先创建备份目录（仅首次需要）
mkdir -p data/backups

# 数据库文件
cp data/database.sqlite data/backups/database_$(date +%Y%m%d).sqlite

# 上传文件目录
cp -r data/uploads/ data/backups/uploads_$(date +%Y%m%d)/
```

> 💡 建议设置定时任务，每天自动备份一次。

### 7.3 更新内容

日常内容更新都在管理后台操作，无需修改代码：
- 新增农产品/景点 → AR 内容管理 → 新增
- 修改介绍文字 → AR 内容管理 → 编辑
- 更换 3D 模型 → AR 内容管理 → 编辑 → 重新上传模型
- 下架旧内容 → AR 内容管理 → 取消发布

### 7.4 模型文件压缩

如果上传的 3D 模型太大，影响手机加载速度：

```bash
# 压缩所有已有模型
node scripts/compress-models.js
```

或者在管理后台编辑内容时，系统会自动压缩超过 2MB 的模型。

### 7.5 查看统计数据

管理后台首页仪表盘可看到：
- AR 内容总数
- 各分类数量
- 每个内容的浏览次数

## 8. 常见问题

### Q: 手机扫描二维码后打不开？

- 确保手机和服务器在同一网络（局域网部署时）
- 检查服务器防火墙是否开放了 3000 端口
- 云服务器需在安全组中放行 3000 端口

### Q: AR 识别不出来？

- 标记图片纹理不够丰富 → 更换更复杂的图片
- 光线太暗或反光 → 调整环境光线
- 摄像头距离不合适 → 保持 20-50cm 距离
- 标记图打印后变形 → 确保平整张贴

### Q: 3D 模型不显示？

- 检查模型格式是否为 `.glb` 或 `.gltf`
- 模型文件是否上传成功（在 data/uploads/models/ 目录确认）
- 手机性能太低 → 尝试压缩模型文件

### Q: 服务器重启后数据丢失？

- 如果使用 Docker，确保挂载了 data 目录（`-v $(pwd)/data:/app/data`）
- 定期备份 `data/database.sqlite` 和 `data/uploads/`

### Q: 如何更新项目代码？

```bash
# 如果使用 Git
git pull

# 重新安装依赖（如有新增）
npm install

# 重启服务
docker restart tangkou-ar    # Docker 方式
pm2 restart tangkou-ar       # PM2 方式
```
